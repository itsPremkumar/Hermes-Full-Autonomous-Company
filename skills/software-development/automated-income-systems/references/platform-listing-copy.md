# Platform Listing Copy — Structure & Principles

> Reference for creating production-grade marketplace listings (Gumroad, Fiverr,
> Medium, GitHub Sponsors). Captures the copy structure, tone rules, pricing
> patterns, and checklist used in a real build (Jul 2026 — Prem Autonomous Co).
>
> The actual files live at `revenue/platform-setup/{gumroad,fiverr,medium,github-sponsors}/`.
> Consult them as worked examples before writing new listings.

## Tone & Quality Rules (US-market professional)

- **No AI-sounding fluff.** Every sentence carries weight. Cut qualifiers
  ("cutting-edge", "revolutionary", "game-changing", "game-changer", "next-gen").
- **Use concrete numbers.** "300+ prompts" not "hundreds of prompts."
  "35+ pieces per week" not "lots of content." "200K organic impressions in 90
  days" not "great results."
- **Structure in three descriptive tiers:**
  1. **The Hook** — name the problem and why existing solutions fail
  2. **The Solution** — what you've built, how it works, what's inside
  3. **The Credibility** — social proof, results data, origin story
- **Write paragraph bodies, not feature lists.** Features become bullet points
  AFTER the descriptive paragraphs. The description sells the outcome; the
  bullets confirm the contents.
- **Avoid exclamation points** (maybe one per page maximum). Professional tone
  means declarative sentences, not hype.
- **No grammatical errors.** Read every sentence aloud. US English spelling
  (realize, organize, optimize, color, center, behavior).

## Gumroad Product Listings

### Required Sections per Product
1. **Title** — benefit-driven, includes number/scope (e.g. "300+ Battle-Tested Templates That Convert")
2. **Subtitle** — one sentence that reframes the value (e.g. "Stop guessing what to say. Start closing...")
3. **Description (3 paragraphs)** — Hook / Solution / Credibility, ~150-200 words each
4. **Key Benefits** — 6 bullet points, each starting with a bolded **label**
5. **What's Included** — itemized list with BONUS items called out
6. **Refund Policy** — 30-day, unconditional, stated simply
7. **About the Creator** — 3-4 sentences: experience, current work, philosophy

### Pricing Patterns Used
| Product Type | Price | Rationale |
|-------------|-------|-----------|
| Prompt packs (code + general) | $9–$14 | Low-friction impulse buy for developers |
| Sales prompts | $19 | B2B value — higher willingness to pay |
| Ebook | $19 | Standard solo-author ebook price point |
| Video templates | $29 | Per-pack; easy to add more packs |
| Content blueprint | $47 | Book-length premium informational product |
| CLI tools | Free + $29 Pro | Freemium — free tier builds trust, Pro is one-time |
| Starter kit | $97 | Highest perceived value — "launch a company" |
| CLI automation kit | $49 | Mid-tier utility toolkit for biz owners |

### Refund Policy Template
"Full refund within 30 days. If [specific claim] doesn't [specific outcome], we'll refund every dollar."

### Creator Bio Template
"Prem Kumar [does X] at Prem Autonomous Co. He has been [doing Y] since 2019 and [doing Z] since 2024. He believes [philosophy statement]."

## Fiverr Service Gigs

### Required Sections per Gig
1. **SEO Title** — "I will [verb] your [target] with [differentiator]"
2. **Category** — pick from Fiverr dropdowns (Programming & Tech > AI Development, etc.)
3. **Gig Description** — three sub-sections:
   - **The Pitch** (problem → solution → outcome)
   - **What I Deliver** (specific deliverables with numbers)
   - **Why Me** (credibility, battle-tested claim)
4. **Package Pricing Table** — Basic / Standard / Premium with 10+ feature rows
5. **FAQ** — 4 Q&A pairs addressing objections
6. **Seller Bio** — who you are, what you've done, how you work

### Package Pricing Design Rules
- **Basic** — core deliverable, gets the job done, no frills
- **Standard** — 2x the scope, adds automation/quality-of-life features
- **Premium** — highest scope, includes support/customization/video
- Feature rows: start with what's in ALL tiers, then differentiate
- Use ✓ for included, — for not included, specific values where variable
- Delivery time scales: 2-3 days (basic) → 5-7 days (standard) → 7-14 days (premium)
- Revisions: 1-2 basic → 2-3 standard → Unlimited premium

### Pricing Range Signal
| Service | Low End | High End | Anchor |
|---------|---------|----------|--------|
| Platform setup (one-time) | $99 | $299 | Paperclip + Hermes deployment |
| Content automation | $149 | $399 | Multi-channel, brand voice |
| Custom workflows | $299 | $799 | Discovery + custom development |
| Code review | $79 | $199 | File-size and agent-count tiers |
| Video production | $49 | $149 | Length and revision tiers |

## Medium Content Calendar

### Structure per Article
1. **Title** — keyword-rich, benefit-driven, specific (not vague)
2. **Subtitle** — SEO-friendly expansion, 15-20 words
3. **Target Keyword** — phrase a US reader would search
4. **Target Publication** — specific Medium publication name
5. **Suggested Length** — 1,800–3,500 words
6. **Key Points / Takeaways** — 5-8 bullet points with real substance

### Scheduling Pattern
- 2 articles per week, 10 weeks = 20 articles
- Monday: technical/instructional (targets Better Programming, freeCodeCamp, TDS)
- Thursday: business/strategy (targets The Startup, Entrepreneur's Handbook, Indie Hackers)
- Cross-publish to company blog with canonical link back to Medium

### Publication Targeting
| Publication | Best For | Submission Tip |
|-------------|----------|----------------|
| The Startup | Entrepreneurship stories | Open to all writers, highest editorial pick rate |
| Better Programming | Developer tooling, architecture | Technical depth required; code samples help |
| Towards Data Science | AI/ML systems | Heavy technical, real data + architecture diagrams |
| Better Marketing | Content strategy, distribution | Results-focused with metrics |
| freeCodeCamp | Tutorials, guides | Educational value first; must teach something |

## GitHub Sponsors Profile

### Structure
1. **Who I Am** — personal intro with timeline and current work
2. **What I Build** — three categories of open-source work with concrete examples
3. **Why Sponsorship Matters** — what the money funds (no ads, no paywalls framing)
4. **Tier Structure** — 5 tiers ($1/$5/$10/$25/$50) with escalating benefits
5. **Sponsor-Only Repository Map** — table showing what each tier unlocks
6. **FAQs** — common questions about billing, changes, company sponsorship

### Tier Benefit Escalation Pattern
| Tier | Price | Key Differentiator |
|------|-------|--------------------|
| Supporter | $1 | Name in README acknowledgments |
| Backer | $5 | Priority issue response + sponsor Discord |
| Advocate | $10 | Private prompt repo + pre-release access |
| Patron | $25 | Monthly group call + production configs |
| Partner | $50 | Monthly 1-on-1 + direct line + custom builds |

### Copy Pattern for Benefits
Each tier: "All [lower tier] benefits + [new differentiator]."
Benefits at one tier are inherited by all higher tiers.

## Pre-Publish Checklist

Before publishing any platform listing, verify:

- [ ] Title contains the primary keyword naturally
- [ ] Subtitle/headline reframes the problem, not just describes the product
- [ ] Three-paragraph structure: Hook → Solution → Credibility
- [ ] Every promise is backed by a specific number or data point
- [ ] Bullet features are outcome-oriented, not feature-speccy
- [ ] Pricing tables (Fiverr) have clear escalation logic
- [ ] Refund policy is stated (products) or revisions defined (services)
- [ ] No "cutting-edge", "game-changing", "revolutionary", "next-gen" or similar buzzwords
- [ ] US spelling (color, center, organize, optimize, behavior)
- [ ] Read aloud for cadence and natural flow
